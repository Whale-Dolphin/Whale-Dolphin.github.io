Scaling the Horizon, Not the Parameters: Reaching Trillion-Parameter Performance with a 35B Agent

中文标题：不扩参数，扩展 Agent 视野：35B Agent 达到万亿参数级表现
信息：LLM Training | score 22 | 2026-06-29, HF 67 upvotes | Lei Bai, Zongsheng Cao, Yang Chen, Zhiyao Cui et al.
链接：arXiv | PDF

为什么值得看：今天最值得读。它把“agent scaling”的重点从模型尺寸转到长轨迹、异构能力和 verifier 反馈闭环，方向上很像把训练数据从静态问答推进到可执行任务经验。

摘要翻译：论文提出 Agents-A1，一个 35B MoE agentic model。作者构建长程知识-动作基础设施，把外部知识、动作、观察和验证器结果连成平均 45K token 的轨迹，再用全域 SFT、领域 teacher、domain-routed on-policy distillation 三阶段训练，把六类能力压进一个可部署学生模型。结果显示，在 SEAL-0、IFBench、HiPhO、FrontierScience-Olympiad、MolBench-Bind 等长程 agent benchmark 上，35B 模型能接近或超过若干 1T 级模型。

小爱精评：如果结果可复现，这篇的意义不只是省参数，而是把“长 horizon 数据 + teacher 路由蒸馏”作为 agent 模型扩展路线。需要重点看 benchmark 是否公平、teacher 成本是否被隐含，以及 45K 轨迹里 verifier 的质量上限。

Agentic Abstention: Do Agents Know When to Stop Instead of Act?

中文标题：Agentic Abstention：Agent 知不知道什么时候该停止行动
信息：AI Agents | score 18 | 2026-06-27, HF 120 upvotes | Han Luo, Bingbing Wen, Lucy Lu Wang
链接：arXiv | PDF

为什么值得看：非常贴近真实 agent 产品。很多失败不是不会做，而是不知道任务已经不可解，继续乱搜乱点。

摘要翻译：作者把 agent abstention 定义为序贯决策问题：agent 每一步都要选择回答、放弃，还是继续收集信息。论文在网页购物、终端和问答等场景评测 13 个 LLM-agent 系统、2 个 agent scaffold，共 28,000 多个任务。结果显示，一些 agent 从不该停时不停，另一些停得太晚。CONVOLVE 通过把完整交互轨迹蒸馏成可复用停止规则，在 WebShop 上把 Llama-3.3-70B 的 timely recall 从 26.7 提到 57.4。

小爱精评：这是 agent 可靠性里被低估的一环。真正的产品需要“及时停止 + 解释为什么停止 + 请求用户补充信息”，这篇至少把第一步评测框架搭起来了。

TACO: Tool-Augmented Credit Optimization for Agentic Tool Use

中文标题：TACO：给 agentic tool use 做工具调用级别的信用分配
信息：LLM Training | score 18 | 2026-06-29, HF 15 upvotes | Mingkuan Feng, Jinyang Wu, Hao Gu, Fangrui Lv et al.
链接：arXiv | PDF

为什么值得看：工具调用 RL 的核心痛点是“哪个工具调用真正帮了最终答案”。这篇给了一个不依赖外部 judge 的 credit assignment 方案。

摘要翻译：TACO 是面向 code-tool multimodal agent 的 GRPO 变体。它用 Differential Answer-Probe Reward 插入 probe token，比对模型在有/无某次工具结果时的答题变化，把每次工具调用标为有用、误导或无效；再用 Outcome-Gated Advantage Routing 把最终答案奖励分配给真正负责的推理片段。实验显示模型准确率提升，同时学会减少无效工具调用。

小爱精评：这篇很实用，因为它把工具调用的奖励从“最后对不对”拆到过程贡献。关键问题是 probe 是否会改变推理分布，以及 answer checker 覆盖不了开放任务时怎么办。

TUA-Bench: A Benchmark for General-Purpose Terminal-Use Agents

中文标题：TUA-Bench：通用终端使用 Agent Benchmark
信息：AI Agents | score 17 | 2026-06-26, HF 44 upvotes | Shoufa Chen, Luyuan Wang, Xuan Yang, Zhiheng Liu et al.
链接：arXiv | PDF

为什么值得看：它把 terminal agent 从 coding 扩到文档、邮件、网页信息、科研工程软件，更接近真实工作流。

摘要翻译：TUA-Bench 有 120 个真实终端任务，覆盖五类任务：常规数字活动、文档编辑、邮件管理、实时网页信息检索，以及 PhD 级专家共同设计的科学/工程工作流。每个任务都有确定性环境和 execution-based scoring。最强 frontier agent（Claude Code + Claude Opus 4.8 max reasoning effort）总体 65.8%，不同任务轨道仍有明显差距。

小爱精评：这类 benchmark 对 Codex/Claude Code 一类 CLI agent 很重要。它提醒我们 terminal agent 不只是写代码，还要会管理文件、查网页、处理邮件和专业软件。

MuseBench: Benchmarking Intent-Level Audiovisual Arts Understanding in MLLMs

中文标题：MuseBench：评测 MLLM 是否理解影音艺术中的创作意图
信息：Audio LLM | score 20 | 2026-06-29 | Yuxuan Fan, Gyusik Seo, Jing Hao, Jaemin Cho et al.
链接：arXiv | PDF

为什么值得看：它把多模态评测从“看到了什么”推进到“为什么这样表达”，对视频/音频模型很有参考价值。

摘要翻译：MuseBench 包含 4,016 个问题，覆盖电影、静态视觉艺术、舞台表演和游戏艺术，数据来自 1 万多个带专业评论的视频 essay。评测目标不是识别物体或镜头，而是判断模型能否理解构图、沉默、节奏、叙事等创作选择背后的意图。28 个 SOTA MLLM 零样本评测中，最佳模型只有 48.29% 准确率，明显低于人类专家的 87.18%。

小爱精评：这是一个更接近“审美/导演意图理解”的 benchmark。短板也很明显：艺术理解本身主观，数据从 video essay 蒸馏可能会带来评论风格偏置。适合跟踪多模态模型的高层语义能力，不适合当作单一综合排名。

Orca: The World is in Your Mind

中文标题：Orca：以 Next-State Prediction 训练通用世界基础模型
信息：其他值得关注 | score 17 | 2026-06-29, HF 64 upvotes | Yihao Wang, Yuheng Ji, Mingyu Cao, Yanqing Shen et al.
链接：arXiv | PDF

为什么值得看：世界模型方向的大体量工作，125K 小时视频和 160M event annotations 的规模值得关注。

摘要翻译：Orca 学习统一的 world latent space，不再只做 next-token、next-frame 或 next-action，而是围绕 Next-State-Prediction 建模状态转移。训练分为 unconscious learning（连续视频中的密集自然状态转移）和 conscious learning（语言描述事件与 VQA 监督下的稀疏有意义转移）。冻结 backbone 后，轻量 decoder 可读出文本生成、图像预测和 embodied action。

小爱精评：概念上很漂亮，像是在给视频、语言、动作找共同的状态空间。需要重点看数据标注质量、状态定义是否真正统一，以及下游 readout 是否证明了 latent 的因果/物理泛化，而不只是大规模视频预训练收益。

SafePyramid: A Hierarchical Benchmark for In-context Policy Guardrailing

中文标题：SafePyramid：面向上下文策略的分层 Guardrail 评测
信息：其他值得关注 | score 17 | 2026-06-29, HF 3 upvotes | Jiacheng Zhang, Haoyu He, Sen Zhang, Shen Wang et al.
链接：arXiv | PDF

为什么值得看：现实产品里的安全规则通常是应用自定义 policy，而不是固定 taxonomy。这篇正好评测这个难点。

摘要翻译：SafePyramid 包含 1,000 段多轮对话、10 个领域、3,000 个应用特定政策，总计 61,699 条自然语言规则。评测分为 L0 单条规则理解、L1 规则依赖推理、L2 完整新 policy framework 适配。10 个 frontier LLM 和 5 个可配置 guardrail 系统表现都不理想，最佳模型在 L0/L1/L2 完整识别违规规则的比例分别只有 54.0%、35.3%、12.9%。

小爱精评：结论很直接：现在的 guardrail 还不擅长“读一份新规则并稳定执行”。这对企业 agent、客服、医疗/金融产品尤其重要。

Bridging VideoQA and Video-Guided Agentic Tasks via Generalized Keyframe Extraction

中文标题：从 VideoQA 到视频指导 GUI Agent：任务驱动关键帧抽取
信息：AI Agents | score 18 | 2026-06-28, HF 21 upvotes | Sunqi Fan, Qingle Liu, Runqi Yin, Meng-Hao Guo et al.
链接：arXiv | PDF

为什么值得看：视频理解和 agent 操作之间的桥接越来越重要，这篇抓住了“关键帧选择”这个低调但很关键的瓶颈。

摘要翻译：论文提出 VG-GUIBench，评测 MLLM GUI agent 能否看视频教程后完成对应 GUI 任务。同时提出 TASKER 关键帧抽取算法，结合任务相关性与场景变化挑选信息帧。TASKER 在 VideoQA 与 video-guided agentic task 上均带来提升，在 EgoSchema fullset 和 NExT-QA 上分别超过最佳 baseline 2.0% 和 1.8%。

小爱精评：对低 token 预算的视频 agent 很有价值。提升幅度不算夸张，但它指出一个可靠方向：视频 agent 的瓶颈经常不是大模型本身，而是输入证据选择。
