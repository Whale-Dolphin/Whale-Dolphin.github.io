Embodied World Models / Robotics

RynnWorld-Teleop: An Action-Conditioned World Model for Digital Teleoperation

  • arXiv: 2607.06558 | PDF
  • 作者: Haoyu Zhao, Xingyue Zhao, Hangyu Li, Biao Gong, Kehan Li et al.
  • 日期: 2026-07-07
  • 预筛分: 21 | HuggingFace: 67 upvotes

一句话: RynnWorld-Teleop 用动作条件生成式世界模型替代真实机器人遥操作，把操作者手部姿态流变成可迁移到不同机器人本体的 imitation learning 轨迹。

精评: 这篇的关键不是单纯「视频生成更真」，而是把机器人数据采集的瓶颈从物理硬件、场地和操作者时间中解耦出来。操作者不再直接控制真实机器人，而是用 hand-pose stream 驱动 robot-centric world model，从单张参考图生成高保真的第一视角视频；姿态流本身又作为 embodiment-agnostic action label，可以通过 retargeting 转到目标机器人。最有价值的点是它把生成过程蒸馏到单次推理，在单 H100 上做到 40+ FPS，这意味着它不是离线数据增强玩具，而开始接近可交互的数据引擎。如果 zero-shot Sim2Real 和真实数据增强的结论能稳定复现，这条路线会明显改变 dexterous manipulation 的数据成本结构。

中文摘要翻译: 扩展机器人学习需要大量、多样的轨迹数据，但当前数据采集受限于物理遥操作：每个示范都绑定操作者时间、具体硬件和工作空间。作者提出 digital teleoperation，用生成式世界模型替代真实机器人，从而把数据采集从物理约束中解耦。在该框架中，操作者的手部姿态流驱动以机器人为中心的生成式世界模型，从单张参考图合成高保真第一视角视频。记录下来的姿态流作为与机器人本体无关的动作标签，可通过标准 retargeting 转移到任意目标机器人，由此得到完整的状态-动作轨迹。作者实现的 RynnWorld-Teleop 集成深度感知骨架条件、人到机器人渐进训练的视频 Diffusion Transformer，以及流式自回归蒸馏，把生成过程压缩成单次推理，在单张 H100 上实现 40+ FPS 的实时交互式生成。仅用 RynnWorld-Teleop 生成数据训练的策略，在多种灵巧双臂任务上实现有效的 zero-shot Sim2Real；把数字遥操作数据加入真实数据集也能稳定提升成功率。

RynnWorld-4D: 4D Embodied World Models for Robotic Manipulation

  • arXiv: 2607.06559 | PDF
  • 作者: Haoyu Zhao, Xingyue Zhao, Siteng Huang, Xin Li, Deli Zhao et al.
  • 日期: 2026-07-07
  • 预筛分: 17 | HuggingFace: 72 upvotes

一句话: RynnWorld-4D 把未来 RGB、深度和光流放进统一 diffusion 过程里生成，用 4D 动态表征服务机器人闭环动作预测。

精评: 这篇和 RynnWorld-Teleop 是一组很强的组合拳。2D 视频 world model 对机器人来说常常差一层：它能预测「看起来会发生什么」，但不一定捕捉到低层控制需要的几何结构和运动场。RynnWorld-4D 用 RGB-DF，也就是同步 RGB、depth、optical flow，把外观、3D 结构和时间运动捆在一起建模。更实用的是它不要求部署时反复 denoise，而是让 inverse dynamics head 直接消费内部 4D representation，单次 forward 输出动作。254.4M frames 的 Rynn4DDataset 也说明这类 embodied world model 正在进入大规模数据工程阶段。

中文摘要翻译: 开放世界中的机器人操作不仅需要识别场景外观，还要预测其 3D 结构在交互下如何运动。作者认为同步 RGB、深度和光流，也就是 RGB-DF，是一种有物理基础的表征，能捕捉场景底层 4D 动态。相比 2D 像素视频，这种多模态协同把视觉外观、几何结构和时间运动对齐，使表征空间更接近机器人系统所需的低层末端执行器动作，从而缩小世界预测与策略学习之间的距离。作者提出 RynnWorld-4D，一个从单张 RGB-D 图像和语言指令出发，在统一 diffusion 过程中共同生成未来 RGB 帧、深度图和光流的生成模型。它采用三分支架构，结合跨模态注意力和逐帧 3D RoPE，确保外观、几何和运动一致演化。作者还构建超过 2.544 亿帧的 Rynn4DDataset 1.0，并提出 RynnWorld-4D-Policy，用 inverse dynamics head 单次前向读取内部 4D 表征，绕过昂贵的多步 denoising，闭环输出机器人动作。实验显示该方法在真实灵巧双臂操作任务上达到 SOTA，尤其擅长空间精度和时间协调要求高的任务。

From Foundation to Application: Improving VLA Models in Practice

  • arXiv: 2607.06403 | PDF
  • 作者: Wei Wu, Fangjing Wang, Fan Lu, He Sun, Shi Liu et al.
  • 日期: 2026-07-07
  • 预筛分: 16 | HuggingFace: 11 upvotes

一句话: LingBot-VLA 2.0 从数据、动作空间和预测式动态建模三方面改进 VLA，让模型更接近真实机器人部署。

精评: 这篇偏实践路线，价值在于把 VLA 从实验室 manipulation benchmark 往真实平台推。作者整理约 60,000 小时数据，其中 50,000 小时是覆盖 20 种机器人配置的轨迹，另有 10,000 小时 egocentric human video；动作空间也扩到头、腰、移动底盘和灵巧手，而不是只处理机械臂末端。future prediction 被当成 proxy task，并结合视频表征和深度估计提供语义与几何先验。这类论文不一定在方法上最花哨，但对判断 VLA 落地需要哪些工程补丁很有参考价值。

中文摘要翻译: 尽管 VLA foundation model 已有进展，实验室条件与真实应用之间的差距仍阻碍其实际部署。作者提出 LingBot-VLA 2.0，在三类功能上改进 LingBot-VLA。第一，提升跨任务和跨本体泛化：重构数据处理流水线，整理约 60,000 小时预训练数据，包括覆盖 20 种机器人配置的 50,000 小时机器人轨迹，以及 10,000 小时第一视角人类视频。第二，在双臂硬件平台之外扩展动作空间，支持头部、腰部、移动底盘和灵巧手自由度，使机器人能处理更复杂的实际任务。第三，引入预测式动态建模以提升时间推理，把未来预测作为代理任务，并利用视频表征模型提供语义先验、深度估计模型提供几何线索。GM-100 benchmark 的 generalist 设置验证了这些改动的收益；得益于覆盖全身自由度的预训练数据，LingBot-VLA 2.0 在两个机器人平台上展现出强跨本体长程移动操作能力。

Image2Sim: Scaling Embodied Navigation via Generative Neural Simulator

  • arXiv: 2607.05765 | PDF
  • 作者: Zihan Wang, Seungjun Lee, Yinghao Xu, Gim Hee Lee
  • 日期: 2026-07-07
  • 预筛分: 16

一句话: Image2Sim 从 posed RGB-D 序列构建实时神经仿真环境，把真实扫描的视觉质量和合成仿真的规模性结合起来。

精评: embodied navigation 的老问题是：真实扫描数据逼真但规模小，传统合成环境可扩展但 sim-to-real gap 大。Image2Sim 的拆法比较聪明，先用 feed-forward feature Gaussian 做 3D 空间锚定，再用 Geometry-Aware One-Step Pixel Flow 把稀疏 noisy projection 变成高质量 panoramic RGB-D observation。它还把视频和图像批量转成近 20K 个可交互场景，并合成 1000 万级导航样本。如果这个 pipeline 稳定，神经仿真会从渲染替代品变成 embodied agent 的数据生产线。

中文摘要翻译: embodied navigation 希望构建能理解多模态目标、在 3D 空间中推理并可靠到达真实目标位置的 agent，但进展受限于缺乏可扩展、高保真且有物理基础的交互环境。真实扫描数据视觉逼真但规模有限，合成仿真器更易扩展却常有较大 sim-to-real gap。作者提出 Image2Sim，一个从 posed RGB-D 图像序列构建高质量交互环境的实时神经仿真框架。核心思想是把 3D 空间锚定与照片级观测合成解耦：场景构建阶段，Image2Sim 用 feed-forward feature Gaussian 模型单次前向将 RGB-D 观测提升为 3D feature-Gaussian 表征；渲染阶段，Geometry-Aware One-Step Pixel Flow 模型把稀疏、噪声较大的 Gaussian projection 转成高质量 panoramic RGB-D 观测。Image2Sim 还作为自动 embodied 数据引擎，从视频和图像构建近 20K 个交互场景，合成超过 1000 万条导航训练样本。完全在这些神经环境中训练的导航模型，在主要 benchmark 上显著提升，并能有效 zero-shot 迁移到真实世界。

Multimodal / Web Agents

SearchEyes: Towards Frontier Multimodal Deep Search Intelligence via Search World Simulation

  • arXiv: 2607.05943 | PDF
  • 作者: Zhengbo Jiao, Yiming Cheng, Yilei Jiang, Kaituo Feng, Rui Huang et al.
  • 日期: 2026-07-07
  • 预筛分: 21 | HuggingFace: 14 upvotes

一句话: SearchEyes 用 typed knowledge graph 构建可复现搜索世界，让多模态搜索 agent 的训练数据、环境和 step-level reward anchor 来自同一套结构。

精评: 多模态 deep search agent 的难点是结构断裂：数据合成时有的 metadata 到环境里丢了，环境又依赖外部搜索引擎导致不可复现，RL 奖励通常只有整条轨迹的稀疏信号。SearchEyes 的贡献是把 Wikidata5M 的视觉-知识交集做成 search world，并用 Perception-Knowledge Chains 采样受约束的多跳路径，保留每一跳实体 metadata。这样同一套结构既定义了环境，也定义了过程奖励锚点。HaPO 用这些锚点做 step-level credit assignment，不需要额外训练 process reward model。对开源多模态搜索 agent 来说，这种可复现的模拟搜索世界比继续调用真实搜索引擎更适合系统训练和消融。

中文摘要翻译: 训练多模态搜索 agent 进行多跳推理仍然困难，原因在于现有流水线在训练数据、搜索环境和奖励信号之间存在结构断裂：合成出的结构 metadata 会被丢弃，环境依赖不可复现的外部搜索引擎，RL 奖励也往往停留在轨迹级稀疏信号。作者提出 SearchEyes，用 typed knowledge graph 作为模拟搜索世界的骨架，统一这三部分。他们提出 Perception-Knowledge Chains，在 Wikidata5M 的视觉-知识交集上采样受约束的多跳路径，保留 hop-level 实体 metadata；这些 metadata 同时定义自包含搜索世界和 step-level reward anchors。作者还提出 Hop-Anchored Policy Optimization，复用这些锚点进行逐步 credit assignment，无需单独训练 process reward model。在六个多模态知识密集 benchmark 上，SearchEyes 在开源多模态搜索 agent 中达到 SOTA，SearchEyes-27B 平均比最强开源 baseline 高 6.2 分。

WebRetriever: A Large-Scale Comprehensive Benchmark for Efficient Web Agent Evaluation

  • arXiv: 2607.06118 | PDF
  • 作者: Wei Dong, Tianyu Fu, Zhe Yu, Hanning Wang, Anyang Su et al.
  • 日期: 2026-07-07
  • 预筛分: 19

一句话: WebRetriever 用 800 个网站、1550 个任务和三类评测协议，更细地评估 web agent 的导航、知识辅助交互和端到端信息提取能力。

精评: 这篇是 AgentGym2 之后又一个「把 agent 评测拉回真实任务」的工作。很多 web agent benchmark 只看是否导航到目标页面，或者只用 screenshot 做 judge，容易漏掉查询语义、筛选条件、信息提取和后续任务完成质量。WebRetriever 的规模和领域覆盖更接近真实部署，还提出 NavEval，用更丰富的交互上下文做 LLM-as-Judge。它最重要的结论是导航成功和真实应用有效性之间有明显断层，这对产品评测很关键：只测 click path 会高估 agent。

中文摘要翻译: 随着 web agent 在自动任务执行中展现能力，如何稳健评估其导航和任务完成表现成为关键问题。现有 benchmark 有三个局限：规模不足、领域多样性有限，难以评估跨领域泛化；主流 LLM-as-Judge 方法无法充分捕捉细粒度交互语义，尤其是精确查询和筛选操作；多数 benchmark 强调导航成功，却忽略真实部署所需的端到端任务要求。作者提出 WebRetriever，一个包含 800 个网站和 1550 个任务的大规模 benchmark，覆盖消费、专业和企业等领域及多种用户意图。作者还提出 NavEval，一个利用视觉截图之外丰富交互上下文的 LLM-as-Judge 框架，并在多个人类判断数据集上达到 SOTA 对齐。WebRetriever 设置了三种互补协议：导航能力、知识辅助交互、以及带信息提取的端到端任务完成。实验显示，不同协议下性能差异很大，说明导航成功本身不足以预测真实应用效果。

Light-Omni: Reflex over Reasoning in Agentic Video Understanding with Long-Term Memory

  • arXiv: 2607.05511 | PDF
  • 作者: Chang Nie, Jiaju Wei, Junlan Feng, Chaoyou Fu, Caifeng Shan
  • 日期: 2026-07-06
  • 预筛分: 18 | HuggingFace: 19 upvotes

一句话: Light-Omni 用全局 multimodal script 和 parametric latent state 替代重型迭代推理，让长视频 agent 以更低延迟完成记忆检索和响应。

精评: 长程视频理解 agent 很容易变成「侦探式」循环：反复搜索、聚合证据、再决定动作，准确但慢且耗显存。Light-Omni 的判断是，这种重推理很多时候是在弥补全局上下文不足和 retrieval 语义错位。它维护一个有限大小的 global state，把 episodic memory 层级合并成 multimodal script；再基于这个全局上下文生成 parametric latent state，直接驱动 action 和 retrieval embedding。结果是比 M3-Agent 平均高 2.4% accuracy，同时 12.1 倍加速、GPU memory efficiency 提升 2.6 倍。对实时视频助理或长期监控类 agent 来说，这种「先组织记忆，再反射式响应」比每次都深推理更现实。

中文摘要翻译: Agentic video understanding 让模型通过长期记忆自主处理并响应连续、长程的多模态流。但高级视频 agent 往往依赖「侦探式」迭代推理来控制动作、搜索证据和聚合信息，带来很高成本和延迟。作者认为，这类重型推理主要是在补偿全局上下文不足和检索语义错位。论文提出 Light-Omni，一个用于反射式、轻量视频理解的多模态 agent 框架。它通过双重上下文状态在单次前向中即时构建所需上下文。首先，系统维护一个 global state，即从 episodic memory 持续整合得到的有限大小 multimodal script，作为 Light-Omni 的全局上下文；通过层级合并，它既保留近期细节又总结过去事件。其次，在该全局上下文条件下，模型生成 parametric latent state，直接驱动自主动作并生成 retrieval embedding，以极低延迟完成响应。得益于这种耦合设计，Light-Omni 避免迭代推理，同时实现语义对齐检索和反射式响应。实验显示它在多个视频 benchmark 上有效，平均比 M3-Agent 高 2.4% accuracy，速度提升 12.1 倍，GPU memory efficiency 提升 2.6 倍，并可作为记忆系统提升现有 MLLM 的性能和效率。

TurnOPD: Making On-Policy Distillation Turn-Aware for Efficient Long-Horizon Agent Training

  • arXiv: 2607.05804 | PDF
  • 作者: Yuhang Zhou, Kai Zheng, Haoling Li, Dengyun Peng, Can Xu et al.
  • 日期: 2026-07-07
  • 预筛分: 14 | HuggingFace: 10 upvotes

一句话: TurnOPD 把长程 agent 的 on-policy distillation 做成 turn-aware budgeting，减少无效尾部 rollout，并让深层决策轮次获得足够 KL 监督。

精评: OPD 对 agent 训练很自然：学生在自己的轨迹上行动，再向强教师对齐。但长程任务里，完整 rollout 往往浪费在尾部弱监督上，trajectory-level KL 又容易被浅层 token 主导，前几步对齐后后面真正决策的 turn 反而训练不够。TurnOPD 用两个预算控制器解决：一个根据 turn statistics 自适应决定 rollout depth，另一个把 loss 从 token-level 逐步转向 turn-balanced。它的工程意义是清楚的：在相同 wall-clock budget 下，把训练算力集中到更有信息量的决策轮次。

精评: 数学推理评测长期偏英语和中文，这会让 reasoning model 的能力看起来比真实全球使用场景更均衡。PluraMath 的贡献不是提出新模型，而是补评测盲区：18 种 underrepresented languages、6 个语系、native speakers 人工校验翻译，再用 27 个 reasoning LLM 做跨规模测试。结论也很符合直觉但需要数据支撑：高资源语言和低资源语言之间仍有持续差距，较好结果往往和 instruction-following 能力更强相关。对多语 reasoning 后训练来说，这类数据集能暴露「会做数学」和「能按本地语言理解题意」之间的差别。

中文摘要翻译: 数学推理已经成为评估和调优 reasoning LLM 的核心任务，但现有 benchmark 严重偏向高资源语言，英语和中文主导预训练语料和评测套件。PolyMath 数据集是重要进展，但也只覆盖 18 种高资源语言。为弥补这一缺口，作者提出 PluraMath，把 PolyMath 扩展到额外 18 种代表性不足的语言，覆盖 6 个语系，从中资源到极低资源设置不等。数据集通过人工 curated pipeline 构建，由母语者仔细校验预计算翻译。作者用 PluraMath 评测 27 个 reasoning LLM，覆盖小、中、大模型和闭源 ensemble，分析 SOTA 模型在不同语言条件下的多语数学推理能力。细粒度分析确认，高资源语言和代表性不足语言之间仍存在持续数学推理性能差距，较强结果通常与更好的 instruction-following 能力相关。作者开源数据集、数据获取 pipeline 和评测框架，以降低 underrepresented communities 构建多语 benchmark 的门槛。
