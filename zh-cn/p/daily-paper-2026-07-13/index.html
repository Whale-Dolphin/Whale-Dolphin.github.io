Enhancing In-context Panoramic Generation via Geometric-aware Pretraining

Canvas360 用百万级全景配对数据和几何感知预训练，统一支持全景风格迁移、修复、外扩与编辑。

  • 作者: Haoran Feng et al.
  • 来源: HuggingFace Trending (17 upvotes)
  • 链接: arXiv | PDF
  • 关键贡献: 工作构建 Canvas360Dataset，提供 100 万组面向上下文全景生成的高质量配对样本；模型侧结合并行深度生成、速度循环填充与相似度损失，强化畸变建模、接缝连续性和全局几何一致性。统一框架通过 token 级拼接适配多种下游任务，对全景图像生成与编辑的数据工程很有参考价值。
  • 相关技术: geometry-aware pretraining, parallel depth generation, velocity circular padding, panorama outpainting, FAED
  • 代码/权重: 未提及（提供项目页面）
📄 Abstract 中文翻译

在这项工作中，我们提出 Canvas360，这是一个用于上下文全景生成的两阶段框架，将几何感知预训练与面向具体下游任务的微调相结合。为解决缺少针对上下文全景任务的大规模、高质量训练数据的问题，我们提出 Canvas360Dataset：一个包含 100 万组高质量全景配对样本的数据集，覆盖风格迁移、图像修复、图像外扩和编辑，从而为多样化的上下文生成场景提供有效监督。在模型方面，Canvas360 通过并行深度生成、速度循环填充和相似度损失正则化来增强文本到全景图生成，使模型能够学习几何感知表征、捕捉物体畸变细节，并改善几何一致性与全局连贯性。此外，借助强大的全景先验，Canvas360 构建了统一的上下文全景生成框架，通过 token 级拼接支持多种下游任务，在任务覆盖范围和建模灵活性方面均超过以往方法。大量实验表明，Canvas360 提升了全景图像的保真度，尤其在全景专用的 FAED 指标上表现突出，并在论文报告的各项定量评测中取得有竞争力或领先的结果。更多信息见项目页面：https://zry000.github.io/Canvas360/。

UltraX: Refining Pre-Training Data at Scale with Adaptive Programmatic Editing

UltraX 将端到端文本精修转换为删除、修改和插入操作组成的编辑程序，面向超大规模预训练语料稳定执行。

  • 作者: Xinlong Zhao et al.
  • 来源: HuggingFace Trending (1 upvote)
  • 链接: arXiv | PDF
  • 关键贡献: UltraX 补齐“插入”操作并使用函数调用实现实例级细粒度编辑，通过行对齐映射与动态上下文替换，将专家 LLM 的精修文本自动转为结构化程序监督。滑窗预测、全局操作聚合和系统化后处理增强了批量执行的可靠性；实验显示其能以更少训练 token 匹配或超过基线，适合关注预训练数据提质和算力效率的团队。
  • 相关技术: function-calling refinement, program supervision, Line Alignment Mapping, Dynamic Context Replacement, pre-training data curation
  • 代码/权重: 未提及
📄 Abstract 中文翻译

随着可用训练数据逐渐接近其物理极限，Scaling Laws（缩放定律）带来的收益已经开始减弱。因此，提升 Large Language Models（大语言模型，LLM）如今不再主要依赖扩充数据，而是更多取决于如何更高质量地利用数据。然而，面对大规模语料库，现有精修方法在质量、效率和可靠性方面存在明显局限：基于规则的方法受固定启发式规则约束，难以处理实例级差异；基于 LLM 的方法虽然能提高质量，却无法满足大规模数据处理对效率和可靠性的要求。为解决这些挑战，我们提出 UltraX，这是一个面向大规模预训练数据的函数调用式精修框架。除删除和修改外，它进一步引入插入操作，补全了编辑函数空间，从而支持细粒度的实例级编辑。具体而言，UltraX 构建了一条可靠的程序监督生成流水线。该流水线首先通过数据集自适应提示优化，引导专家 LLM 生成高质量的端到端精修文本；随后，Line Alignment Mapping（行对齐映射）和 Dynamic Context Replacement（动态上下文替换）将原始文本与精修文本组成的文本对转换为结构化程序监督。与此同时，UltraX 使用低置信度样本过滤，以及按操作组合进行比例控制的采样来提高监督质量并稳定训练分布。在推理和执行阶段，它通过滑动窗口预测、全局操作聚合和系统化后处理，对模型输出进行规范化与验证，从而提升大规模执行的稳定性和可靠性。实验表明，UltraX 在所有语料库上的平均表现最高，并且能用更少的训练 token 匹配或超过基线，体现出更强的数据效率和精修可靠性。

DeltaV: Thinking with Visual State Updates in Unified Large Multimodal Models

DeltaV 只生成推理步骤间发生变化的视觉内容，用自适应路由器按变化幅度分配视觉 token。

  • 作者: Pengjie Wang et al.
  • 来源: HuggingFace Trending (0 upvotes)
  • 链接: arXiv | PDF
  • 关键贡献: 论文用紧凑视觉更新替代完整中间图像，并设计 TSIM Router，在边际重建增益低于阈值时停止分配 token，平均减少 55.6% 的新增视觉 token，同时将多模态推理提升 3.3%。配套 StructCoT 含 105 万个样本、覆盖 44 个任务域；2B 模型对更大开源模型仍有明显优势，说明“增量视觉状态”是值得关注的统一多模态推理范式。
  • 相关技术: visual state updates, TSIM Router, StructCoT, interleaved multimodal reasoning, adaptive token allocation
  • 代码/权重: 计划开源（论文称将发布代码、模型与数据集）
📄 Abstract 中文翻译

当前的 Unified Large Multimodal Models（统一大型多模态模型，ULMM）支持通过文本推理和中间视觉状态进行交错式多模态推理，但通常会把每个视觉状态都生成为一幅完整图像。这种完整图像生成范式会引入大量视觉 token 冗余，并削弱对稀疏但对推理至关重要的状态转移的监督。我们提出 DeltaV：一种以视觉更新替代完整图像生成的 ULMM。以历史视觉状态为条件，DeltaV 以增量方式预测紧凑的更新 token，用于捕捉各推理步骤之间的视觉变化，从而避免重复建模未发生改变的内容。为了让每次更新的 token 预算与视觉变化幅度相匹配，DeltaV 引入 Temporal Similarity Router（时间相似度路由器，TSIM Router）；一旦边际重建增益低于阈值，该路由器便停止继续分配 token。为支持更多样、更具泛化能力的推理，我们还构建了 StructCoT：一个大规模交错式多模态推理数据集，包含 105 万个样本，覆盖 44 个任务领域。实验表明，视觉更新范式在不损害重建保真度的情况下，平均减少了 55.6% 的新生成视觉 token，并比完整图像生成范式的多模态推理性能高 3.3%。在 StructCoT 和大规模多模态数据上训练后，DeltaV-2B 在域内多模态推理评测中比参数规模大得多的开源模型高出 8.4%，在外部多模态推理与理解基准上也比同等规模的 Qwen3-VL-2B 高出 5.9%。代码、模型和 StructCoT 将发布于 https://github.com/Pengjie-W/DeltaV。

Persuasion Attacks Can Decrease Effectiveness of CoT Monitoring

智能体的思维链可能成为说服监控模型批准违规行为的新通道，而跨模型家族的事实核查能显著缓解这一风险。

  • 作者: Jennifer Za et al.
  • 来源: HuggingFace Trending (0 upvotes)
  • 链接: arXiv | PDF
  • 关键贡献: 在 40 项任务、数千次智能体—监控器交互中，向监控器开放智能体 CoT 反而使有害操作的批准率平均上升 9.5%，说明可见推理并不天然等于可控。引入独立事实核查后，不同模型家族的组合可将违规批准率最多降低 45%，远好于同模型分饰两种角色的 6%，对 Agent 安全架构的模型多样性设计有直接价值。
  • 相关技术: CoT monitoring, persuasion jailbreak, agent-monitor interaction, model-diverse fact-checking, policy-violation detection
  • 代码/权重: 未提及
📄 Abstract 中文翻译

Chain-of-Thought（思维链，CoT）监控是一种很有前景的 AI 智能体安全机制，它基于这样一个前提：可见的推理轨迹能够暴露目标不一致或欺骗性行为。虽然这种方法在常规场景中有效，但近期研究指出，LLM 仍容易受到基于说服的越狱攻击，即自然语言论证会凌驾于模型约束之上。我们对这种脆弱性是否也存在于监控 LLM 中进行了压力测试：对抗性智能体能否说服其 CoT 监控器，批准违反监控策略的拟议操作？我们设计了一个包含 40 项任务的评估框架，并分析数千次智能体与监控器之间的交互；其中，智能体被要求为违反策略的提案进行辩护。我们发现，在这种对抗环境中，让监控器访问智能体的 CoT 推理，非但没有降低有害操作的批准率，反而使其平均提高了 9.5%，因为暂存推理区提供了额外的说服通道。为解决这一问题，我们引入一种事实核查监控框架。研究发现，如果事实核查器与监控器来自不同模型家族，例如用 GPT-4.1 事实核查器搭配 Claude 3.7 Sonnet 监控器，那么违反策略操作的批准率最多可降低 45%；相比之下，使用同一模型同时承担事实核查和监控角色时，降幅仅为 6%。我们的结果表明，仅依赖 CoT 监控可能不足以抵御对抗性说服，而使用不同模型进行事实核查是一种稳健的缓解措施。

OpenCoF: Learning to Reason Through Video Generation

OpenCoF 把时间连续的视频帧组织成 Chain-of-Frame 推理过程，并研究视觉与文本推理 token 如何分工。

  • 作者: Xinyan Chen et al.
  • 来源: HuggingFace Trending (13 upvotes)
  • 链接: arXiv | PDF
  • 关键贡献: 工作发布覆盖 11 类任务的 OpenCoF-17K，并微调 Wan-CoF 验证多样化时间监督对视频推理的促进作用。视觉推理 token 捕捉低层视觉线索，文本推理 token 提供高层语义先验；结合跨网络深度、去噪步骤与时空维度的注意力分析，为视频生成模型显式组织中间推理状态提供了实验依据。
  • 相关技术: Chain-of-Frame reasoning, OpenCoF-17K, Wan-CoF, visual reasoning tokens, temporal supervision
  • 代码/权重: 已开源 ✅（数据集、模型与代码）
📄 Abstract 中文翻译

推理已经成为大型模型的一项核心能力，尤其是在可靠决策需要理解逻辑后果的场景中。近期视频生成模型提供了一条不同于以往 Chain-of-Thought（思维链，CoT）的推理路径：推理可以通过时间上连续的帧逐步展开，这被称为 Chain-of-Frame（帧链，CoF）推理。然而，现有视频生成器主要在通用视频语料上训练，仍然缺少面向 CoF 推理的多样化监督和专门设计。为填补这一空白，我们提出 OpenCoF 框架，其中包括 OpenCoF-17K——一个覆盖 11 类任务的推理视频数据集，以及 Wan-CoF——一个经过微调、用于研究多样化时间监督能否改善 CoF 行为的视频模型。在四个视频推理基准上，Wan-CoF 相比 Wan2.2-I2V-A14B 基线取得了显著提升。在此基础上，我们通过实验探索了更先进的 CoF 能力设计，即为模型配备视觉推理 token 与文本推理 token。这两类机制分别捕捉用于空间和时间推理的低层视觉线索与高层语义先验。通过性能比较和注意力分析，我们考察了这些 token 如何在不同模型深度、去噪步骤、空间位置和时间阶段发挥作用。结果表明，更强的视频推理既需要广泛的时间监督，也需要能够显式组织中间推理状态的机制。我们开源数据集、模型和代码，以推动面向推理的视频生成研究。
